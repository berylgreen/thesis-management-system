## 二、代码变更清单

### 📂 后端变更 (7 个文件)

#### 1. 新增异常类 (3 个文件)

**`backend/src/main/java/com/thesis/exception/NotFoundException.java`**
- 用途: 404 资源不存在异常

**`backend/src/main/java/com/thesis/exception/ForbiddenException.java`**
- 用途: 403 权限拒绝异常

**`backend/src/main/java/com/thesis/exception/BadRequestException.java`**
- 用途: 400 非法请求异常

#### 2. 全局异常处理器

**`backend/src/main/java/com/thesis/exception/GlobalExceptionHandler.java`**
- 关键修复: 使用 `ResponseEntity` 映射 HTTP 状态码
- 影响范围: 所有 REST API 错误响应
- 代码片段:
```java
@ExceptionHandler(NotFoundException.class)
public ResponseEntity<Result<Void>> handleNotFound(NotFoundException e) {
    return ResponseEntity.status(HttpStatus.NOT_FOUND)
            .body(Result.error(e.getMessage()));
}
```

#### 3. 安全工具类

**`backend/src/main/java/com/thesis/util/SecurityUtil.java`**
- 方法: `getCurrentUserId()`, `getCurrentUserRole()`
- 用途: 从 JWT SecurityContext 提取当前用户信息

#### 4. 服务层增强

**`backend/src/main/java/com/thesis/service/ThesisService.java`**
- 新增方法: `compareVersionsWithAuth(Long v1Id, Long v2Id, Long userId, String role)`
- 关键逻辑:
  1. 认证校验 (null 拒绝)
  2. 参数校验 (版本ID合法性)
  3. 资源加载 (版本、论文)
  4. 跨论文阻断
  5. 角色权限校验 (STUDENT/TEACHER 白名单)
  6. ContentHash 短路
  7. 文件对比执行

#### 5. 控制器层改造

**`backend/src/main/java/com/thesis/controller/DiffController.java`**
- 变更: 调用 `ThesisService.compareVersionsWithAuth` 替代直接调用 DiffUtil
- 增加: SecurityUtil 调用获取 userId 和 role

#### Critical 问题修复记录

1. **HTTP 状态码未映射**
   - 发现者: Codex 后端审查
   - 影响: API 契约违反,前端无法正确处理错误
   - 修复: GlobalExceptionHandler 返回 ResponseEntity

2. **认证绕过风险**
   - 发现者: Codex 安全扫描
   - 影响: 未认证用户可能访问任意论文
   - 修复: compareVersionsWithAuth 方法开头校验 userId/role 非 null

3. **角色权限缺失**
   - 发现者: Codex 权限分析
   - 影响: ADMIN 等角色默认通过校验
   - 修复: 添加 STUDENT/TEACHER 白名单,其他角色显式拒绝

4. **重复数据库查询**
   - 发现者: Codex 性能分析
   - 影响: 每次对比多执行 2 次 SELECT
   - 修复: 直接使用已加载的 v1/v2 对象的 filePath 属性

5. **前端内存泄漏**
   - 发现者: Gemini 前端审查
   - 影响: 重复打开对比 Dialog 导致内存累积
   - 修复: onUnmounted 中移除 resize 监听器

## 五、部署指南

### 🚀 前置条件

- Node.js 18+
- Java 17+
- MySQL 8.0
- Redis 7.0

### 📦 前端部署步骤

```bash
cd frontend

# 1. 安装依赖 (包含 diff2html)
npm install

# 2. 构建生产版本
npm run build

# 3. 部署 dist 目录到 Nginx
# 或使用 Docker:
docker-compose up -d frontend
```

### 🔧 后端部署步骤

```bash
cd backend

# 1. 编译 (Maven 自动下载 Java Diff Utils)
mvn clean package -DskipTests

# 2. 运行
java -jar target/thesis-system.jar

# 或使用 Docker:
docker-compose up -d backend
```
