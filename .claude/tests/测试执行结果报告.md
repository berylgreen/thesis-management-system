### 前端测试修复

**修复方案 1**: 使用 `flushPromises`

安装依赖:
```bash
npm install @vue/test-utils --save-dev
```

修改测试:
```javascript
import { flushPromises } from '@vue/test-utils'

it('应该在加载失败时显示错误提示', async () => {
  vi.spyOn(thesisApi, 'getDiff').mockRejectedValue({
    response: { data: { message: '版本不存在' } }
  })

  wrapper = mount(VersionComparer, {
    props: mockProps,
    global: { plugins: [ElementPlus] }
  })

  await flushPromises() // 等待所有 Promise 完成
  await wrapper.vm.$nextTick()

  const alert = wrapper.find('.el-alert')
  expect(alert.exists()).toBe(true)
})
```

**修复方案 2**: 使用 setTimeout 延迟

```javascript
it('应该在加载失败时显示错误提示', async () => {
  // ...
  await new Promise(resolve => setTimeout(resolve, 100))
  await wrapper.vm.$nextTick()
  // ...
})
```

**修复方案 3**: 监听组件内部状态

```javascript
it('应该在加载失败时显示错误提示', async () => {
  // ...
  await wrapper.vm.$nextTick()

  // 等待 isLoading 变为 false
  while (wrapper.vm.isLoading) {
    await new Promise(resolve => setTimeout(resolve, 10))
  }

  const alert = wrapper.find('.el-alert')
  expect(alert.exists()).toBe(true)
})
```
