## 🔴 问题描述

### 问题现象
执行 `mvn test` 时出现编译失败:
```
[ERROR] Failed to execute goal org.apache.maven.plugins:maven-compiler-plugin:3.11.0:compile
(default-compile) on project thesis-system: Fatal error compiling: 无效的目标发行版本: 17
```

### 环境信息
- **操作系统**: Windows 10 (中文版)
- **Maven**: 已安装并配置
- **Java 版本**: Java 25.0.1 (而非项目要求的 Java 17)
- **编码**: UTF-8 (已配置)

### 根本原因
1. **JDK 版本不匹配**: 项目配置为 Java 17,但系统安装的是 Java 25
2. **Maven 编译器插件问题**: 在 Windows 中文环境下,`maven.compiler.release` 参数解析异常
3. **Spring Boot Parent POM**: 自动设置了 `maven.compiler.release=17`,覆盖了手动配置

---

## ✅ 解决方案

### 方案 A: 安装 JDK 17 (推荐)

**步骤**:
1. 下载 JDK 17: https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html
2. 安装到英文路径 (如 `C:\Java\jdk-17`)
3. 设置环境变量 `JAVA_HOME=C:\Java\jdk-17`
4. 更新 PATH: 将 `%JAVA_HOME%\bin` 添加到前面
5. 验证: `java -version` (应显示 17.x.x)
6. 重新执行测试:
   ```bash
   cd E:\7antigravity\test16\backend
   mvn clean test -Dtest=ThesisServiceTest,DiffControllerTest
   ```

**优点**:
- ✅ 完全匹配项目配置
- ✅ 避免潜在的版本兼容性问题
- ✅ 符合生产环境标准

**缺点**:

## 💡 技术说明

### 为什么 Java 25 不能编译 Java 17 目标?

理论上 Java 25 应该支持生成 Java 17 字节码,但在 Windows 中文环境下,Maven 编译器插件的参数传递可能出现编码问题:

1. **正常情况**: `javac -source 17 -target 17 File.java`
2. **实际执行**: 由于中文系统编码,参数可能被误解析
3. **错误信息**: "无效的目标发行版本: 17"

这是一个已知的 Windows + Maven + JDK 的组合问题,最可靠的解决方案是:
- 使用项目配置的 JDK 版本 (Java 17)
- 或使用 IDE 的内置编译器 (会自动处理这些问题)
