## 一、快速开始

### 后端测试

```bash
cd backend

# 1. 执行所有测试

# 2. 生成覆盖率报告 (HTML)

# 3. 查看覆盖率报告

# 4. 仅执行版本对比相关测试
```

## 四、持续集成 (CI)

### 4.1 GitHub Actions 配置 (示例)

创建文件 `.github/workflows/test.yml`:

```yaml
name: Tests

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  backend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up JDK 17
        uses: actions/setup-java@v3
        with:
          java-version: '17'
          distribution: 'temurin'
      - name: Run tests
        run: mvn clean test -f backend/pom.xml
      - name: Generate coverage report
        run: mvn jacoco:report -f backend/pom.xml
      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v3
        with:
          files: backend/target/site/jacoco/jacoco.xml
          flags: backend

  frontend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install dependencies
        run: npm ci --prefix frontend
      - name: Run tests
        run: npm test --prefix frontend -- --run
      - name: Generate coverage
        run: npm run coverage --prefix frontend
      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v3
        with:
          files: frontend/coverage/coverage-final.json
          flags: frontend
```

## 五、常见问题

### Q1: 后端测试失败 "No tests found"

**原因**: 测试类命名不符合 Maven Surefire 规范
**解决**:
- 测试类必须以 `Test` 结尾 (如 `ThesisServiceTest`)
- 或以 `Test` 开头 (如 `TestThesisService`)
- 或以 `TestCase` 结尾

### Q2: 前端测试报错 "Cannot find module '@/api/thesis'"

**原因**: Vitest 未配置路径别名
**解决**: 检查 `vite.config.js` 中的 `resolve.alias` 配置

### Q3: Jacoco 报告显示覆盖率 0%

**原因**: 未执行测试或 jacoco-maven-plugin 配置错误
**解决**:
```bash
mvn clean test jacoco:report
```

### Q4: Element Plus 组件在测试中报错

**原因**: 缺少全局插件或 Mock
**解决**: 在 `mount()` 时注册插件:
```javascript
import ElementPlus from 'element-plus'

mount(Component, {
  global: {
    plugins: [ElementPlus]
  }
})
```

### Q5: "window.matchMedia is not a function"

**原因**: JSDOM 不支持 `matchMedia`
**解决**: 已在 `src/tests/setup.js` 中 Mock
