## 二、后端测试计划

### 2.1 单元测试 (ThesisService)

**测试类**: `ThesisServiceTest.java`
**框架**: JUnit 5 + Mockito
**覆盖方法**: `compareVersionsWithAuth()`

| 测试用例 | 预期结果 | 优先级 |
|---------|---------|--------|
| 用户未认证 (userId=null) | ForbiddenException | P0 |
| 角色未认证 (role=null) | ForbiddenException | P0 |
| 版本ID为null | BadRequestException | P1 |
| 对比同一版本 | BadRequestException | P1 |
| 版本不存在 | NotFoundException | P1 |
| 跨论文对比 | BadRequestException | P0 |
| 学生对比自己论文 | 返回差异列表 | P0 |
| 学生对比他人论文 | ForbiddenException | P0 |
| 教师对比任意论文 | 返回差异列表 | P0 |
| 管理员角色访问 | ForbiddenException | P0 |
| ContentHash相同 | 返回空列表 | P1 |
| 文件读取失败 | RuntimeException | P2 |

**Mock 对象**:
- ThesisMapper
- ThesisVersionMapper
- DiffUtil

### 2.2 集成测试 (DiffController)

**测试类**: `DiffControllerTest.java`
**框架**: SpringBootTest + MockMvc
**覆盖端点**: `GET /api/diff/compare`

| 测试用例 | HTTP 状态码 | 响应体 | 优先级 |
|---------|-----------|--------|--------|
| 学生对比自己论文 | 200 | data: [差异列表] | P0 |
| 学生对比他人论文 | 403 | error: "无权访问他人论文" | P0 |
| 教师对比任意论文 | 200 | data: [差异列表] | P0 |
| 未认证用户 | 403 | error: "用户未认证" | P0 |
| 版本不存在 | 404 | error: "版本不存在" | P1 |
| 参数缺失 | 400 | error: "版本ID不能为空" | P1 |
| 跨论文对比 | 400 | error: "不能对比不同论文的版本" | P1 |

**Mock 策略**:
- Mock ThesisService (隔离业务逻辑)
- Mock Security Context (模拟 JWT 认证)

## 五、测试执行计划

### 5.1 本地执行

**后端**
```bash
cd backend
mvn clean test                        # 执行所有测试
mvn test -Dtest=ThesisServiceTest     # 执行单个测试类
mvn jacoco:report                     # 生成覆盖率报告 (需配置 jacoco)
```

**前端**
```bash
cd frontend
npm test                              # 运行所有测试
npm run test:ui                       # 可视化测试界面
npm run coverage                      # 生成覆盖率报告
```

## 七、风险与限制

### 风险

1. **文件系统依赖**: DiffUtil 需要真实文件,单元测试需 Mock 文件操作
2. **Spring Security 集成**: 集成测试需正确 Mock JWT 认证上下文
3. **diff2html 渲染**: 前端测试难以验证 HTML 渲染结果

### 缓解措施

1. 使用 `@TempDir` (JUnit 5) 创建临时测试文件
2. 使用 `@WithMockUser` 或手动构建 SecurityContext
3. 验证 diff2html 输入数据格式而非 HTML 输出
