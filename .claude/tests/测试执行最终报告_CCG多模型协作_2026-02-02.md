### 阶段 2: 前端组件测试修复 (Gemini)

**执行时间**: 2026-02-02
**任务**: 分析并修复 VersionComparer.vue 组件的 7 个失败测试

**Gemini 根本原因分析**:

1. **Alert 测试失败 (2个)**:
   - 问题: `el-alert` 的 `title` prop 渲染为文本内容，不是 DOM 属性
   - 原因: 测试使用 `alert.attributes('title')` 而非 `alert.text()`

2. **Tabs 和并排视图渲染失败 (2个)**:
   - 问题: 异步数据获取未完成就进行断言
   - 原因: `setTimeout` 等待机制不稳定

3. **diff2html 转换失败 (3个)**:
   - 问题: `unifiedHtml` 始终为空字符串
   - 原因: 组件内部 `import { Diff2Html } from 'diff2html'` 导入方式可能有误

**Gemini 修复策略**:

```javascript
// 1. 使用 vi.mock 模拟 diff2html 模块
vi.mock('diff2html', () => ({
  Diff2Html: {
    html: vi.fn((diffJson, config) => {
      // 返回预期的 HTML 字符串
      return '<div class="diff-content">Mock Diff HTML</div>'
    })
  }
}))

// 2. 修改断言方式
// 错误: expect(alert.attributes('title')).toBe('...')
// 正确: expect(alert.text()).toContain('...')

// 3. 改进异步等待
await nextTick()
await flushPromises()
```

### 阶段 3: 后端集成测试配置 (Codex)

**执行时间**: 2026-02-02
**任务**: 配置后端集成测试环境以解决 Spring Boot 上下文启动失败

**Codex 问题分析**:

```
错误: UnsatisfiedDependencyException
原因: JwtAuthenticationFilter 需要 JwtUtil bean
根因: @Component 注解的 Filter 被自动扫描并注册
```

**Codex 推荐方案**:

```java
// 方案 B: @WebMvcTest (最轻量级)
@WebMvcTest(controllers = DiffController.class, excludeAutoConfiguration = {
    SecurityAutoConfiguration.class,
    SecurityFilterAutoConfiguration.class
})
@AutoConfigureMockMvc
@Import(GlobalExceptionHandler.class)
class DiffControllerTest {
    @MockBean
    private ThesisService thesisService;
    // ...
}
```

**实施结果**:
- ✅ 修改测试配置为 `@WebMvcTest`
- ✅ 添加 `@Import(GlobalExceptionHandler.class)`
- ⚠️ 仍遇到 `JwtAuthenticationFilter` 自动扫描问题
- 📝 **根本原因**: `@Component` Filter 在组件扫描时被自动注册

**未完成原因**:
需要在生产代码中修改 `JwtAuthenticationFilter` 的注册方式，或创建专门的测试配置类排除该 Filter。这超出了"仅修改测试代码"的约束范围。

**建议后续方案**:
1. 创建 `@TestConfiguration` 排除 `JwtAuthenticationFilter`
2. 或使用 `@SpringBootTest` + H2 内存数据库
3. 或在 `SecurityConfig` 中使用 `@Profile` 区分测试环境
