# 论文管理系统 - 用户使用指南

本指南包含 **Web系统操作手册**、**系统启动与维护** 及 **文件命名规范**，旨在帮助用户高效使用论文管理系统。

---

## 第一部分：系统启动与环境依赖

### 1. 环境依赖 (Dependencies)

在运行本系统前，请确保本地环境已安装以下基础软件：

* **Java**: JDK 17+
* **Node.js**: v18+ (推荐 LTS 版本)
* **Docker Desktop**: 必须安装并启动 (用于运行 MySQL 8.0数据库 和 Redis 缓存服务)
* **Maven**: 3.6+ (后端构建工具)

### 2. 启动与停止 (一键脚本)

项目根目录提供了 `dev.ps1` PowerShell 脚本，用于一键管理所有服务（后端、前端、数据库）。

#### ✅ 启动系统

在 PowerShell 终端运行：

```powershell
.\dev.ps1 -Action start
```

* 脚本会自动启动 Docker 容器。
* 后台运行 Spring Boot 后端 (端口 8080)。
* 后台运行 Vue 前端 (端口 3000 或 8888)。
* 启动成功后，请访问浏览器查看。

#### 🛑 停止系统

```powershell
.\dev.ps1 -Action stop
```

* 该命令会安全停止所有相关服务及 Docker 容器。

#### 🔄 重启系统

```powershell
.\dev.ps1 -Action restart
```

#### ℹ️ 查看状态

```powershell
.\dev.ps1 -Action status
```

## 第二部分：Web系统操作手册

系统提供可视化的论文管理界面，支持在线上传、版本对比及历史记录回溯。

### 1. 登录与角色

* **登录地址**: 访问系统首页（例如 `http://localhost`, 具体视部署端口而定）。
* **默认账号**:
  * 若通过文件自动导入创建的账号，默认密码通常为 `admin123`。
  * 系统区分 **学生** 与 **教师** 角色，不同角色看到的菜单略有差异。

### 2. 论文管理 (控制台)

进入「论文管理」菜单：

* **学生试图**: 只能查看自己的论文列表。
  * **新建论文**: 点击右上角“新建论文”按钮，输入标题即可创建空白记录。
* **教师视图**: 可以查看所有学生的论文。
  * **强制更新**: 教师端拥有“强制更新”按钮，点击后系统会扫描 `uploads` 目录，同步最新的文件系统变更到数据库（常用于手动归档文件后）。

### 3. 版本详情与上传

点击论文列表的“查看详情”进入版本管理页：

* **基本信息**: 顶部显示论文标题。
* **版本列表**: 表格展示所有历史版本（V1, V2...），包含文件名、大小、备注及上传时间。
* **上传新版本**:
    1. 点击“上传新版本”按钮。
    2. 选择本地文件（支持 `.docx`, `.pdf`）。
    3. 输入“版本说明”（例如：修正了摘要部分）。
    4. 提交后，系统会自动增加版本号（如 V1 -> V2）。
* **下载文件**: 点击列表右侧的“下载”即可获取对应版本的原始文件。

### 4. 在线版本对比 (核心功能)

系统支持在线查看文档内容及版本差异对比。

* **单版本查看**: 在列表中勾选 **1个** 版本，点击“查看与对比”按钮，即可在线预览文档内容。
* **差异对比**: 在列表中勾选 **2个** 版本（通常选最新的和上一个），点击“查看与对比”。
  * 系统会自动识别 新/旧 版本。
  * 进入对比视图，左侧显示旧版，右侧显示新版，差异点会以高亮标注（新增/删除/修改）。

## 第二部分：文件导入与命名归档规范

为了确保系统能正确识别后台手动放入的文件，请严格遵守以下命名约定。

### 1. 核心命名规范

```text
姓名学号_论文题目_日期.扩展名
```

#### ✅ 正确示例

* `张诗妍124232022025_开题报告_20260206.docx`
* `林雨萱124232022030_基于Vue的毕设系统_20251231.pdf`

#### ❌ 错误示例

* `124232022025_张诗妍_作业.docx` (姓名应在前)
* `张三_论文.docx` (无学号)

### 2. 批量归档流程 (脚本辅助)

适用于批量收集学生文件的情况：

1. **准备**: 将文件放入 `source/sourseFilis` 目录（文件名只需包含学生姓名）。
2. **执行**: 运行 `python scripts/organize_files.py`。
3. **结果**: 脚本自动重命名并整理到 `backend/uploads/{学号}/` 目录。
4. **同步**: 在 Web 端（教师号）点击“强制更新”或重启服务生效。

### 3. 手动归档流程

管理员直接操作服务器文件：

1. 进入 `backend/uploads/`。
2. 创建/进入以 **12位学号** 命名的文件夹（如 `124232022025`）。
3. 将符合命名规范的文件拷贝入内。

## 常见问题 (FAQ)

**Q: 只有 Word 文档支持对比吗？**
A: 是的，目前的在线对比功能主要针对 `.docx` 文档进行优化的内容级对比；PDF 文件仅支持下载或基础预览。

**Q: 为什么上传失败？**
A: 请检查文件名是否包含特殊字符，或文件大小是否超过服务器限制（通常默认为 10MB-50MB）。

**Q: 文件名里的日期有什么用？**
A: 系统会记录文件名里的日期作为版本备注的一部分，但实际版本排序主要依据“导入/上传的物理时间”。
