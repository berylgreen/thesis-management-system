# 版本对比排序逻辑优化实施计划

## 1. 目标
优化版本对比功能的排序与展示逻辑，确保：
- 版本列表中，日期更新的、标号（版本号）更大的版本排在前面。
- 在版本对比界面中，日期较新（或版本号较大）的版本显示在左侧，日期较旧的版本显示在右侧。
- 差异分析逻辑保持正确（即对比“旧版本 -> 新版本”的变化），但 UI 布局满足用户“新左旧右”的习惯。

## 2. 影响范围
- 后端服务：`ThesisService.java` (列表排序)
- 前端视图：`ThesisDetail.vue` (对比参数准备)
- 前端组件：`VersionComparer.vue` (UI 布局调整)

## 3. 执行步骤

### 3.1 后端排序优化
- 修改 `ThesisService.getThesisVersions` 方法。
- 排序规则：`created_at` DESC, `version_num` DESC。

### 3.2 前端对比参数逻辑 (ThesisDetail.vue)
- 在点击“对比”按钮时，对已选择的两个版本进行排序。
- 确定 `oldVersion` 和 `newVersion`。
- 将 `oldVersion` 传给 `version1Id`，`newVersion` 传给 `version2Id`（保持 API 逻辑：旧 -> 新）。

### 3.3 前端 UI 布局调整 (VersionComparer.vue)
- 在分屏显示模式下，将“新版本 (Revised)”的 `el-col` 移动到左侧。
- 将“旧版本 (Original)”的 `el-col` 移动到右侧。
- 在并排文本对比 (Side-by-side) 逻辑中（如果有手动生成的 HTML），确保新版本在左。
- 更新相关的 CSS 类名或标题（如果需要）。

## 4. 风险评估
- **差异方向反转**：如果只是简单交换 ID 而不调整 UI 布局，会导致“增加”显示为“删除”。必须保持 API 调用为 `(旧, 新)`，仅在 UI 上交换位置。
- **并排对比 HTML**：`sideBySideHtml` 是由后端还是前端生成的？如果是前端生成的，需要检查生成逻辑。如果是后端，需要看 API 返回格式。

## 5. 验证标准
1. 上传多个版本（同日、异日）。
2. 版本列表应按时间降序排列。
3. 选择任意两个版本对比。
4. 对比界面左侧标题应为“新版本”，右侧为“旧版本”。
5. 验证差异颜色：新版本中新增的内容应显示为绿色（增加），旧版本中存在但新版本删除的内容应在右侧显示或以红色标记（取决于具体实现）。
