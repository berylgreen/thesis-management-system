# Maven 修复与安装计划

## 1. 现状诊断

- **故障现象**：终端无法识别 `mvn` 命令。
- **环境检查**：`PATH` 中无 Maven 路径。
- **文件扫描**：`C:/Program Files` 及 `Users` 目录下未发现标准安装。发现 `D:` 盘 IntelliJ IDEA 内置 Maven。
- **工具链**：系统已安装 `winget`。

## 2. 解决方案

### 首选方案：Winget 安装 (推荐)

- **命令**：`winget install Maven.Maven`
- **优势**：标准化安装、自动配置环境变量、易于升级。
- **风险**：国内网络环境下 GitHub/CDN 下载可能受阻。

### 备选方案：复用 IntelliJ 内置 Maven

- **路径**：`D:\Program Files\JetBrains\IntelliJ IDEA 2025.3.2\plugins\maven\lib\maven3\bin`
- **操作**：将该路径写入用户/系统环境变量 `PATH`。
- **劣势**：耦合 IDE 版本，卸载 IDE 会导致 Maven 失效。

## 3. 执行步骤

1. 尝试 Winget 安装。
2. 若成功，验证 `mvn -version`。
3. 若失败 (网络超时)，回退至备选方案（配置 PATH 指向 IntelliJ 内置 Maven）。

## 4. 验证标准

- 终端运行 `mvn -version` 返回 Apache Maven 3.x 版本信息。
